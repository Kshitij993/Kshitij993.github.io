Characteristics of Java

  • Simple
  • Object-oriented
  • Compiled
  • Architectural neutral
  • Multi-threaded
  • Garbage collection
  • Robust
  • Extensible language

JAVA has become enormously popular over time and has remained in most of the surveys among top three popular programming languages over two decades. This popularity of JAVA, and its acceptance can be attributed to its features and design. As stated in the Java language white paper by Sun, Java is simple, object-oriented, distributed, interpreted, robust, secure, architecture-neutral, portable, high-performance, multithreaded, and dynamic. Let’s analyze these often-used buzzwords.

Simple:

Java is very simple to learn with quite clean and easily understandable syntax. It is based on C++ programming language(hence easier to switch from C++) . Many complicated but rarely used concepts like operator overloading and explicit pointers, etc, has been removed from Java. It provides Automatic Garbage Collection in Java hence no need to remove unreferenced objects. These are some of the features that makes Java, a simple programming language.

Object-oriented:

Java is a pure object oriented programming language. Without class concept, not a single a program can be written in Java. Organizing software in combination of different types of objects that incorporates data as well as well as behaviour is object-oriented. The methodology of object oriented programming language(OOPs), simplifies the development and maintenance of software by providing some rules. Basic concepts of OOPs are:

  • Object
  • Class
  • Polymorphism
  • Abstraction
  • Encapsulation

Platform Independent:

Java provides WORA(write once run anywhere) feature. Java is a platform independent programming language as it is different from other languages like C,C++, etc, which are compiled into platform specific machines. This platform can be the hardware or software environment where the program runs. Java itself provides-platform based platform. This software-based platform runs on top of other hardware-based platforms. Its components are Java Runtime Environment and Application Programming Interface(API). Java code is compiled by the compiler and converted into bytecode which is then converted into machine language and hence this enables it to run on run on multiple platforms, for example, Windows, Linux, Sun Solaris, Mac/OS, etc.

Security:

Java is known very well for the security it provides. Virus free system can be developed. It is secure as:

  • The concept of explicit pointer is not there.
  • Classloader is a part of JRE which is used to load classes into the JVM dynamically, which adds security by separating the package for the classes of the local file system from those that are imported from network sources.
  • It runs inside a virtual machine sandbox which servers as a protective cover.
  • There is bytecode verier that looks for illegal code that is potential voilater of access right to objects.
  • Security manager that keeps tab on the resources that can be accessed by a class.

Some security can also be provided by an application developer explicitly through SSL, JAAS, Cryptography, etc.

Robust:

Java is strong or robust as:

  • It uses strong memory management.
  • There is automatic garbage collection in java which runs on the Java Virtual Machine to get rid of objects which are not being used by a Java application anymore.
  • There are exception handling and the type checking mechanism in Java. All these points make Java robust.
  • There is bytecode verier that looks for illegal code that is potential voilater of access right to objects.
  • There is a lack of pointers that avoids security problems and make usability simpler.

Portable:

The property of platform independence facilitated by WORA property provides java with the feature of high portability. It facilitates us to carry the Java bytecode to any platform.

Architecture-neutral:

Java is architecture neutral because there are no implementation dependent features, for example, the size of primitive types is fixed. In C programming, int data type occupies 2 bytes of memory for 32-bit architecture and 4 bytes of memory for 64-bit architecture. However, it occupies 4 bytes of memory for both 32 and 64-bit architectures in Java.

Compiled and Interpreted language:

Java is considered to be both a compiled and an interpreted language because its source code is first compiled into a binary byte-code. This byte-code runs on the Java Virtual Machine (JVM), which is usually a software-based interpreter. The use of compiled byte-code allows the interpreter (the virtual machine) to be small and efficient (and nearly as fast as the CPU running native, compiled code).

High-performance:

Java code is faster than other High level languages as Java bytecode is "close" to native code. It is still a little bit slower than a compiled language (e.g., C++). Java is an interpreted language that is why it is slower than compiled languages, e.g., C, C++, etc.

Distributed:

Java is distributed because it facilitates users to create distributed applications in Java. Many systems allow us to run the code remotely written in a domain-specific language, and Java distributes the code that gets easily compatible with the programming environment. RMI and EJB are used for creating distributed applications. This feature of Java makes us able to access files by calling the methods from any machine on the internet.

Multi-threaded:

A thread is like a separate program, executing concurrently. We can write Java programs that deal with many tasks at once by defining multiple threads. Java is a multi-threaded programming language which means we can develop multi-threaded program using Java. A multi-threaded program contains two or more parts that can run concurrently and each part can handle a different task at the same time making optimal use of the available resources specially when your computer has multiple CPUs.

Dynamic:

Java is a dynamic programming language. Dynamic loading of class is supported by it, i.e , it supports loading of class on demand. It also supports functions from its native languages, i.e., C and C++. Objects and members can be created dynamically. Java supports dynamic compilation and automatic memory management (garbage collection).
